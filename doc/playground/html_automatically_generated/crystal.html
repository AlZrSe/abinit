3- Crystalline structure and symmetries

This file gives the settings related to the crystalline structure and the symmetries of the system, to build an input file for ABINIT.

Contents of ABINIT features:

  • 1. Introduction.
  • 2. Tutorial and examples.
  • 3. Related input variables.

 

1. Introduction

The cell may be orthogonal or non-orthogonal. Any kind of symmetries and corresponding sets of k-point can be input, and taken into account in the computation. The crystal structure and the position of the atoms in the unit cell must be specified. Details about the way the crystal structure is defined in ABINIT can be found here.

 

 

2. Tutorials and examples.

  • The lesson 1 deals with the H2 molecule : get the total energy, the electronic energies, the charge density, the bond length, the atomisation energy
  • The lesson 2 deals again with the H2 molecule: convergence studies, LDA versus GGA
  • The lesson 3 deals with crystalline silicon (an insulator): the definition of a k-point grid, the smearing of the cut-off energy, the computation of a band structure, and again, convergence studies ...
  • The lesson 4 deals with crystalline aluminum (a metal), and its surface: occupation numbers, smearing the Fermi-Dirac distribution, the surface energy, and again, convergence studies ...

    • The reader can find many examples of input files in the directory /tests of the ABINIT package.

    Compulsory input variables:

    ... acell [CELL lattice vector scaling]
    ... rprim [Real space PRIMitive translations]

    Useful input variables:

    ... scalecart [SCALE CARTesian coordinates]

    Input variables for experts:

    ... xyzfile [XYZ FILE input for geometry]

    Relevant internal variables:

    ... %rprimd [Real space PRIMitive translations, Dimensional]
